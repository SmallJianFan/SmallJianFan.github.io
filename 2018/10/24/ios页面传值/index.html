ios页面传值

发表于 2018-10-24

传值分类

页面传值基本分为两种：正向传值和反向传值。

（一）初始化传值

初始化传值适用于上一级界面往下一级界面传值ViewControllerA -> ViewControllerB

ViewControllerB界面写一个类似构造函数的方法

.h页面定义一个方法

- (id)initWithUserName:(NSString*)userName WithPassWord:(NSString *)passWord;

.m实现构造方法

@implement ViewControllerB
- (id)initWithUserName:(NSString*)userName WithPassWord:(NSString *)passWord{
    self = [super init];
    if(self){
    _passWord = passWord;
    _userName = userName;
    }
    return self；
}

ViewControllerA调用传值处

ViewControllerB *VCB = [[ViewControllerB alloc]initWithUserName:_userNameText.text WithPassWord:_passWordText.text];

（二）属性传值

首先介绍一下属性传值，属性传值是真的简单，也是最常用到的，属性传值又称为正向传值。
首先有ViewControllerA 和 ViewControllerB，现在我要实现ViewControllerA 跳转到ViewControllerB ，并把ViewControllerA 页面的值传给ViewControllerB 页面(把A页面的一个string传递给B页面)。

// 在ViewControllerA 界面

import “ViewControllerA.h”

import “ViewControllerB.h”

@interface ViewController ()

@property(nonatomic,strong)UITextField *textField;

@property(nonatomic,strong)UIButton *button;

@end

@implementation ViewController

- (void)viewDidLoad {

      [super viewDidLoad];

      _textField = [[UITextField alloc]initWithFrame:CGRectMake(30, 100, 120, 40)];

      _textField.layer.borderColor = [UIColor lightGrayColor].CGColor;

      _textField.layer.borderWidth = 0.5;

      [self.view addSubview:_textField];

      _button = [[UIButton alloc]initWithFrame:CGRectMake(10, 10, 60, 60)];

      [_button setTitle:@"跳转" forState:UIControlStateNormal];

      [_button setTitleColor:[UIColor blackColor] forState:UIControlStateNormal];

     [_button addTarget:self action:@selector(buttonClick) forControlEvents:UIControlEventTouchUpInside];

      [self.view addSubview:_button];


  }

    - (void)buttonClick{

  ViewControllerB * VCB = [[ViewControllerB alloc]init];

  VCB.strB = _textField.text;

  [self presentViewController:VCB animated:YES completion:nil];
   }

@end

// 在ViewControllerB 界面

// 首先要先在该界面定义一个属性（该属性为你要传值的类型的属性），比如说你要传字符串类型，你要先定义一个字符串类型的变量

在.h文件定义

@property (nonatomic,strong)NSString *strB;

@property (nonatomic,strong)UILabel *label;

@property (nonatomic,strong)UIButton *btn;

在.m文件写

 - (void)viewDidLoad {
    [super viewDidLoad];

    self.view.backgroundColor = [UIColor whiteColor];

    _label = [[UILabel alloc]initWithFrame:CGRectMake(30, 100, 120, 40)];

    _label.text = _strB;

    [self.view addSubview:_label];

   _btn = [[UIButton alloc]initWithFrame:CGRectMake(10, 10, 60, 60)];

    [_btn setTitle:@"返回" forState:UIControlStateNormal];

    [_btn setTitleColor:[UIColor blackColor] forState:UIControlStateNormal];

    [_btn addTarget:self action:@selector(buttonClick) forControlEvents:UIControlEventTouchUpInside];

    [self.view addSubview:_btn];
  }

    [self dismissViewControllerAnimated:YES completion:nil];

}

（三）Block传值

block传值（反向传值）

哪个界面需要反向传值在哪个界面定义block

A页面跳转到B页面，B页面在返回A页面的时候，判断是否调用了block，并进行传值..同时，在A页面进行block的回调。。。

总的来说就是，B页面定义block，在B页面写好Block要传的值，在A页面进行回调，这样就会在A页面显示B页面要传的值。

相关代码

@property (nonatomic,copy)void(^block)(NSString *text);

// block最好用copy；个人理解：默认情况下，block会存档在栈中，所以block会在函数调用结束被销毁，在调用会报空指针异常，如果用copy修饰的话，可以使其保存在堆区，它的生命周期会随着对象的销毁而结束的。只要对象不销毁，我们就可以调用在堆中的block。

// B要传文字，他是需要声明block的一方，在.h文件中声明

// B声明，然后调用（这样才能传过去一个数，才叫反向传值）

// 声明一个属性，指向要调用的block

/*

1.驱动方声明一个block属性 驱动方调用block,执行block中得代码

2.被动方要具体实现block<驱动的block属性应该指向被动方实现的block>

*/

if (_block) {

_block(textField.text);

}

（四）代理传值

委托要做的事情：

在想要传值的界面 （B界面）声明委托变量、定义协议方法、调用委托协议

在.h文件声明

@protocol passByValuedDelegate 

- (void)passValue:(NSString *)str;

@end

@interface ViewControllerB : UIViewController

// 声明委托变量

@property (nonatomic,weak)id delegate;

.m 文件 通过委托变量调用委托协议方法

[self.delegate passValue:self.label.text];

在被传值的界面 （A界面） 执行代理协议、实现委托方法

import “ViewController.h”

import “ViewControllerB.h”

@interface ViewController ()

@end

- (void)buttonClick{

ViewControllerB * VCB = [[ViewControllerB alloc]init];

VCB.delegate = self; //遵循代理协议

[self presentViewController:VCB animated:YES completion:nil];

}

// 实现协议方法

- (void)passValue:(NSString *)str{

self.textField.text = str;

}

（五）通知传值（NSNotification）

通知中心传值，可以跨越多个页面传值，一般也是从后面的页面传给前面的页面。当然，正向传值也是OK的。

思路：

第三个界面的值传给第一个界面。

1、在第一个界面建立一个通知中心，通过通知中心，注册一个监听事件

2、在第一个界面中，设置接收到通知的事件（也就是观察者所监听的事件）

3、在第一个界面中的dealloc中，将通知中心remove掉（这一步是必须的，不要忘记哦🙅）

4、在第三个界面中，建立一个通知中心，通过通知中心，发送通知（发送通知的过程就是传值的过程，将要传输的值作为object的值传给第一个界面）

代码：

第一个界面（B界面）

//通知中心是个单例

NSNotificationCenter *notiCenter = [NSNotificationCenter defaultCenter];

//注册一个监听事件。第三个参数的事件名，系统用这个名字来区别不同的事件。

 [notiCenter addObserver:self selector:@selector(receiveNotification:) name:@"notification" object:nil];

//@selector(receiveNotification:)方法，就是接收到通知之后的事件

 - (void)receiveNotification:(NSNotification *)noti
{

self.textField.text = noti.object;

 // NSNotification 有三个属性，name, object, userInfo，其中最关键的object就是从第三个界面传来的数据。name就是通知事件的名字， userInfo一般是事件的信息。

NSLog(@"%@ === %@ === %@", noti.object, noti.userInfo, noti.name);

 }

 // 第一界面中dealloc中移除监听的事件

 - (void)dealloc
{
    // 移除当前对象监听的事件

    [[NSNotificationCenter defaultCenter] removeObserver:self];

}

第三个界面（即要传值的界面 A界面）

// 创建一个通知中心

 NSNotificationCenter *center = [NSNotificationCenter defaultCenter];

  // 发送通知. 其中的Name填写第一界面的Name， 系统知道是第一界面来相应通知， object就是要传的值。 UserInfo是一个字典， 如果要用的话，提前定义一个字典， 可以通过这个来实现多个参数的传值使用。

    [center postNotificationName:@"notification" object:_label.text userInfo:dic];

总结：若A想往B传值，需要在A界面建立一个通知中心，发送消息。B界面建立通知中心，通过通知中心创建监听者，设置接收的信息。在B界面一定要将通知中心remove掉。

注：如果想要实现多个参数的传值可以在传值界面（A界面）定义字典，在B界面通过监听方法字典取值来实现传值。

简单来说，A传值到B，在A创建通知，如果想要多个参数传递，则创建一个字典。B为接收值的界面，所以在B界面注册通知监听者，将通知发送的消息接收。

（六）单例传值

单例传值用起来比较方便，就是需要多两个文件。当然了，如果不创建也能用，也有些是写在APPdelegate.h里的。下面给大家介绍一下单例的基本使用方法：

1、首先创建一个文件，继承NSObject

（1）.h文件中

#import <Foundation/Foundation.h>

NS_ASSUME_NONNULL_BEGIN

  @interface DefaultInstance : NSObject

 @property (nonatomic ,copy)NSString *str;//传值的属性，想传什么值就赋什么值。

 @property (nonatomic, copy)NSString *str1;

 +(instancetype)sharedInstance;//共享单例的方法

 /*如果想要传递多个参数的话，可以创建一个字典，把值存到字典（第一个方法），在获取值的时候只需要获取字典调用第二个方法即可。*/

 - (void)initWithDic:(NSDictionary *)dict;

  - (NSDictionary *)getTheDict;

  @end

 NS_ASSUME_NONNULL_END

（2）.m中实现方法

+(instancetype)sharedInstance{
    static DefaultInstance *shareInfo = nil;
    if (shareInfo == nil) {
        shareInfo = [[DefaultInstance alloc]init];
    }
    return shareInfo;
}

- (void)initWithDic:(NSDictionary *)dict{

[DefaultInstance sharedInstance].str = dict[@"textField1"];

[DefaultInstance sharedInstance].str1 = dict[@"textField2"];

}

- (NSDictionary *)getTheDict{

    NSDictionary *dic = @{@"textField1":self.str,@"textField2":self.str1};

    return dic;
}

2、传值的界面要实现的方法（传值界面如果要实现多个传值的话，肯定是要实现字典的创建的，即定义的第一个方法）

（1）导入头文件：

#import "DefaultInstance.h"

（2）给单例赋值

NSDictionary *diction = @{@"textField1":self.textField.text,@"textField2":self.textField1.text};

[[DefaultInstance sharedInstance]initWithDic:diction];

3、获取传值的界面要实现的方法（先获取到存到单例里的字典，然后从字典里获取相应的值）

（1）导入头文件：

#import "DefaultInstance.h"

（2）拿到单例的值，这样就传值成功了

NSDictionary *dic = [[DefaultInstance sharedInstance]getTheDict];

_label.text = dic[@"textField1"];

_label1.text = dic[@"textField2"];

4、同理，单例也可以反向传值，第二个控制器在给单例赋值之后，在第一个控制器获取单例值就行（写在viewWillAppear:方法里）。

总结：在用字典传递时，字典的key一定要保持一致，否则会crash。那么一共涉及到了四个用字典的地方，到底哪个跟哪个要保持一致呢？当然，全部是一致的肯定不会出错。大家可以仔细看一下，我这里就直接告诉大家喽。传递值的字典和方法传入的字典的key值一定要保持一致。在获取值和取值的时候一定要保持一致，不要随意设定key值哦~

（七）NSUserDefault传值

对没有跳转关系的两个controller进行传值（类似弄个sqlite数据库传值了，或写个文件传值，区别单例在于，应用被kill后再次启动还有值，而单例重启应用后，就木有值了）

ViewControllerA -> ViewControllerB

ViewControllerA类

//赋值保存

 [[NSUserDefaults standardUserDefaults]setValue:self.textField.text forKey:@"textField"];

[[NSUserDefaults standardUserDefaults]setValue:self.textField1.text forKey:@"textField1"];

//写入到NSUserDefaults

[[NSUserDefaults standardUserDefaults]synchronize];

ViewControllerB类

_label.text = [[NSUserDefaults standardUserDefaults]objectForKey:@"textField"];
